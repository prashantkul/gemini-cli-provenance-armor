============================================================
  LOO SCORING INTERPRETATION
============================================================

  The LOO (Leave-One-Out) method runs 3 passes per tool call:

  Pass 1: P(action | User + History + Untrusted)  — full context
  Pass 2: P(action | History + Untrusted)          — user removed
  Pass 3: P(action | User + History)               — untrusted removed

  Span influence = Pass 1 - Pass 3
    (how much removing the untrusted file changes the score)

  User influence = Pass 1 - Pass 2
    (how much removing the user instruction changes the score)

  In all 8 attack scenarios:
    - Span influence >> 0  (removing README drops score massively)
    - User influence ≈ 0   (removing user barely changes anything)
    → The README is the dominant cause, not the user
    → BLOCKED

  Attack span influence range:  87 to 136
  Attack user influence range: -28 to 5
  Ratio (span/|user|):         3x to 50x dominance
